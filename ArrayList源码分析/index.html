ArrayList源码解析

发表于 2019-03-18 | 分类于 集合源码 |

综述

ArrayList是一个动态数组，它是线程不安全的，允许元素为null。

其底层数据结构依然是数组，它实现了List, RandomAccess, Cloneable, java.io.Serializable接口，其中RandomAccess代表了其拥有随机快速访问的能力，ArrayList可以以O(1)的时间复杂度去根据下标访问元素。

优缺点：应为ArrayList其底层数据结构是数组，它占据了一块连续的内存空间，所以它有着数组的缺点，就是空间效率不高，同样，由于数组的内存连续，可以根据下标以O1的时间读写(改查)元素，因此时间效率很高。

注意：
ArrayList的扩容操作是一个性能消耗比较大的地方。在后续博客中，源码中的add方法基本都有确认是否需要扩容，而删，改，查等都有越界检查，所以不多赘述。

构造方法（初始化）

构造方法共有三种，分别是无参，初始化容量，以及初始化直接复制过来的集合。

需要注意的是，Collection.toArray()以及Arrays.copyOf()方法的使用率十分高，现在对Arrays.copyOf()说明一下，它用于复制指定的数组内容以达到扩容的目的，该方法对不同的基本数据类型都有对应的重载方法。

Arrays.copyOf(U[] original, int newLength, Class<? extends T[]> newType)，

  • @param original 要复制的数组
  • @param newLength 副本的长度
  • @param newType 副本的类

它在底层进行数组拷贝时调用的是本地方法 System.arraycopy()；Arrays.copyOf()方法返回的数组是新的数组对象，原数组对象仍是原数组对象，不变，该拷贝不会影响原来的数组。

public static native void arraycopy(Object src, int srcPos,
Object dest, int destPos,
int length);

  • @param src：源对象
  • @param srcPos：源数组中的起始位置
  • @param dest：目标数组对象
  • @param destPos：目标数据中的起始位置
  • @param length：要拷贝的数组元素的数量

通过源码可以很容易的看出每次添加数据之前都会判断是否达到容量的限度，如果达到，就要扩容。

add(E e)方法，直接就是在数组末尾加上一个数据，而add(int index, E element)方法添加数据时，其原理是，把index位置空出来，所有数据往后推移，把指定数据放在index位置。

addAll()方法类似第三种构造函数，采用System.arraycopy()方法来添加数据，两种重载不同的是一个是末尾添加，一个是index位置开始添加，但实现方法相似。

接下来说扩容，当数据达到初始化时数组长度就需要扩容，以避免数组越界：

扩容的流程：在add类方法中，先对添加数据的size计算，看添加这些数据后容量是否超过ArrayList底层数组的长度，如果未超过，什么都不做，如果超过，则需要扩容，它在原数组的基础上扩容一半，也就是变成ArrayList容量变成原来的1.5倍（使用位移的方法可以增加性能），如果这个容量还不能容纳添加的数据，那么就用能容纳的最小的数量（add后的容量）。最后采用Arrays.copyOf(elementData, newCapacity)拷贝，扩容，构建一个新数组。

add、addAll。 先判断是否越界，是否需要扩容。 如果扩容， 就复制数组。 然后设置对应下标元素值。

数据删除

remove的核心思想也类似add，移除index位置数据，采用System.arraycopy()方法，将index位置后面的数据往前移动一步。

移除指定元素，先判断是否存在，如果存在则找到他的index，采用上述方法移动数据。

移除一个指定集合内的数据，采用的是覆盖的方法，采用两个指针，一个用来迭代，一个用来记录删除元素后指针的位置，针对ArrayList数组判断每个数据是否存在指定集合内，如果不存在两个数组的指针共步向前移动一步，如果存在，则w不动，r++，这样需要删除的元素就会被覆盖。
